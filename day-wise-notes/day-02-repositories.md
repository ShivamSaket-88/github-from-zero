# Day 2 – GitHub Repositories

## What is a Repository?
A repository (repo) is a place where a project’s files, code, and
documentation are stored together.

## What does a repository contain?
- Source code
- README file
- Documentation
- Issues and pull requests history

## When should you create a repository?
- When starting a new project
- When documenting learning
- When collaborating with others

## How to create a repository?
1. Click on “New Repository”
2. Enter repository name
3. Choose public or private
4. Add README (recommended)
5. Click Create

## Where is a repository used?
- Personal projects
- College assignments
- Open-source projects
- Team collaboration

## Key learning from Day 2
- One project should have one repository
- README explains the purpose of the repo
- Repositories keep work organized
