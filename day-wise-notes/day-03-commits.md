## What is a Commit?
A commit is a snapshot of changes made to files in a repository.
It helps track progress and maintain history.

## When should you commit?
- After completing a small task
- After fixing a bug
- After updating documentation

## Why commits are important?
- Keep a record of changes
- Help revert to previous versions
- Show contribution history

## What makes a good commit?
- Small and focused changes
- Clear commit message
- Related to one task only

## Examples of good commit messages
- Add Day 3 commit notes
- Fix typo in README
- Update repository structure

## Key learning from Day 3
- Commit frequently but meaningfully
- One commit should represent one change
- Clear commit messages matter
